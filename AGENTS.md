# Repository Instructions

## Language policy

All repository content should be written in English.

This applies to:

- Source code
- Identifiers when naming is being introduced or changed
- Code comments
- Markdown documentation
- GitHub Actions workflows
- Examples and templates
- AI-generated changes

## Change policy

- Keep documentation aligned with workflow behavior
- Prefer concise, explicit wording
- Do not introduce new Portuguese text into code, docs, or comments
- When editing existing files, translate touched Portuguese content to English
